# LogiEat OS — Database Schema (MySQL 8 / MariaDB)

> 2026-06-26 · Pair with [`ARCHITECTURE.md`](./ARCHITECTURE.md) · [`PRD.md`](./PRD.md)
> **Engine:** MySQL 8 / MariaDB (bundled in **Laragon**, GUI = HeidiSQL). All tables `InnoDB` + `utf8mb4`.
> **Multi-tenancy rule:** every business table carries `company_id`; every query filters by it.
> **IDs:** `CHAR(36)` UUIDs **generated by the app** (Laravel `HasUuids` / Go `google/uuid`) —
> not auto-increment, so ids aren't guessable. Times stored UTC.

## 1. ERD (textual)
```
companies 1──* users
companies 1──* subscriptions
companies 1──* orders
companies 1──* routes 1──* route_assignments *──1 orders
users(courier) 1──* routes
companies 1──* current_locations / location_history
companies 1──* messages
companies 1──* notifications
```

> MySQL has no `CREATE TYPE` — enums are **inline `ENUM(...)`** on the column. Values for
> `food_category` and `spoilage_risk` must match `app.py` exactly.

## 2. Tables

### companies (the tenant)
```sql
CREATE TABLE companies (
  id              CHAR(36) NOT NULL PRIMARY KEY,
  name            VARCHAR(160) NOT NULL,
  owner_name      VARCHAR(120) NOT NULL,
  phone           VARCHAR(32) NOT NULL,
  email           VARCHAR(160) NOT NULL,
  catering_code   VARCHAR(32) NOT NULL,                 -- e.g. 'DBC-7F3A9K' (random hash, NOT serial)
  depot_address   VARCHAR(255),
  depot_lat       DOUBLE,
  depot_lng       DOUBLE,
  vehicle_capacity_default INT NOT NULL DEFAULT 100,
  subscription_status ENUM('inactive','active','expired','cancelled') NOT NULL DEFAULT 'inactive',
  created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uq_companies_email (email),
  UNIQUE KEY uq_companies_code (catering_code)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### users (owner / admin / courier — all tenant-scoped)
```sql
CREATE TABLE users (
  id              CHAR(36) NOT NULL PRIMARY KEY,
  company_id      CHAR(36) NOT NULL,
  role            ENUM('owner','admin','courier') NOT NULL,
  status          ENUM('pending','active','rejected','suspended') NOT NULL DEFAULT 'active', -- couriers start 'pending'
  name            VARCHAR(120) NOT NULL,
  email           VARCHAR(160),
  phone           VARCHAR(32),
  password_hash   VARCHAR(255) NOT NULL,
  vehicle_plate   VARCHAR(20),                          -- courier
  vehicle_capacity INT,                                 -- courier (nullable → company default)
  fcm_token       VARCHAR(255),
  approved_by     CHAR(36),
  approved_at     DATETIME,
  created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uq_users_company_email (company_id, email),
  KEY idx_users_company_role (company_id, role, status),
  CONSTRAINT fk_users_company FOREIGN KEY (company_id) REFERENCES companies(id) ON DELETE CASCADE,
  CONSTRAINT fk_users_approver FOREIGN KEY (approved_by) REFERENCES users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### subscriptions / payments
```sql
CREATE TABLE subscriptions (
  id              CHAR(36) NOT NULL PRIMARY KEY,
  company_id      CHAR(36) NOT NULL,
  plan            VARCHAR(40) NOT NULL,                 -- 'pro_monthly' | 'pro_yearly'
  amount          DECIMAL(12,2) NOT NULL,
  method          VARCHAR(20) NOT NULL,                 -- 'qris' | 'va' | 'ewallet' (mock gateway)
  status          ENUM('inactive','active','expired','cancelled') NOT NULL DEFAULT 'active',
  period_start    DATETIME NOT NULL,
  period_end      DATETIME NOT NULL,
  paid_at         DATETIME,
  created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  KEY idx_subs_company (company_id, status),
  CONSTRAINT fk_subs_company FOREIGN KEY (company_id) REFERENCES companies(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### orders
```sql
CREATE TABLE orders (
  id              CHAR(36) NOT NULL PRIMARY KEY,
  company_id      CHAR(36) NOT NULL,
  code            VARCHAR(24) NOT NULL,                 -- human ref e.g. '#1021' (unique per company)
  recipient_name  VARCHAR(160) NOT NULL,
  recipient_phone VARCHAR(32),
  menu_name       VARCHAR(200) NOT NULL,
  food_category   ENUM('Santan','Basah','Kering'),      -- drives Q10 spoilage in app.py
  quantity        INT NOT NULL DEFAULT 1,               -- = SchoolNode.demand (pax)
  address         VARCHAR(255) NOT NULL,
  maps_link       VARCHAR(512),
  latitude        DOUBLE NOT NULL,
  longitude       DOUBLE NOT NULL,
  deadline_at     DATETIME,                             -- batas antar
  time_window_minutes DOUBLE,                           -- optional manual override
  status          ENUM('pending','assigned','picked_up','delivered','cancelled') NOT NULL DEFAULT 'pending',
  spoilage_risk   ENUM('critical','high','medium','low'),-- cached from last optimize (display)
  created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uq_orders_company_code (company_id, code),
  KEY idx_orders_company_status (company_id, status),
  CONSTRAINT fk_orders_company FOREIGN KEY (company_id) REFERENCES companies(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### routes (one optimized run for one courier)
```sql
CREATE TABLE routes (
  id              CHAR(36) NOT NULL PRIMARY KEY,
  company_id      CHAR(36) NOT NULL,
  courier_id      CHAR(36) NOT NULL,
  status          ENUM('assigned','in_progress','completed','cancelled') NOT NULL DEFAULT 'assigned',
  depot_lat       DOUBLE NOT NULL,                      -- snapshot at dispatch time
  depot_lng       DOUBLE NOT NULL,
  temperature     DOUBLE,                               -- sent to app.py
  total_distance_km   DOUBLE,
  total_time_minutes  DOUBLE,
  model_type      VARCHAR(60),                          -- 'a2c_spoilage_aware_v2' | heuristic
  risk_summary    JSON,                                 -- {critical,high,medium,low}
  created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  started_at      DATETIME,
  completed_at    DATETIME,
  KEY idx_routes_company_courier (company_id, courier_id, status),
  CONSTRAINT fk_routes_company FOREIGN KEY (company_id) REFERENCES companies(id) ON DELETE CASCADE,
  CONSTRAINT fk_routes_courier FOREIGN KEY (courier_id) REFERENCES users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### route_assignments (ordered stops within a route)
```sql
CREATE TABLE route_assignments (
  id              CHAR(36) NOT NULL PRIMARY KEY,
  company_id      CHAR(36) NOT NULL,
  route_id        CHAR(36) NOT NULL,
  order_id        CHAR(36) NOT NULL,
  sequence        INT NOT NULL,                         -- 1..N from app.py
  estimated_minutes   DOUBLE,
  distance_km     DOUBLE,
  spoilage_risk   ENUM('critical','high','medium','low'),
  minutes_until_spoilage DOUBLE,
  status          ENUM('pending','arrived','delivered','skipped') NOT NULL DEFAULT 'pending',
  pod_photo_url   VARCHAR(512),                         -- proof of delivery
  pod_lat         DOUBLE,
  pod_lng         DOUBLE,
  delivered_at    DATETIME,
  UNIQUE KEY uq_ra_route_seq (route_id, sequence),
  KEY idx_ra_route (route_id, sequence),
  CONSTRAINT fk_ra_company FOREIGN KEY (company_id) REFERENCES companies(id) ON DELETE CASCADE,
  CONSTRAINT fk_ra_route FOREIGN KEY (route_id) REFERENCES routes(id) ON DELETE CASCADE,
  CONSTRAINT fk_ra_order FOREIGN KEY (order_id) REFERENCES orders(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### current_locations (one row per courier — fast map reads)
```sql
CREATE TABLE current_locations (
  courier_id      CHAR(36) NOT NULL PRIMARY KEY,        -- upsert on every GPS ping
  company_id      CHAR(36) NOT NULL,
  route_id        CHAR(36),
  latitude        DOUBLE NOT NULL,
  longitude       DOUBLE NOT NULL,
  heading         FLOAT,
  speed_kmh       FLOAT,
  recorded_at     DATETIME NOT NULL,
  KEY idx_curloc_company (company_id),
  CONSTRAINT fk_curloc_company FOREIGN KEY (company_id) REFERENCES companies(id) ON DELETE CASCADE,
  CONSTRAINT fk_curloc_courier FOREIGN KEY (courier_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
-- Upsert: INSERT ... ON DUPLICATE KEY UPDATE latitude=VALUES(latitude), ...
-- (Replaces Postgres "DISTINCT ON" pattern: read = single-row lookup per courier.)
```

### location_history (append-only GPS trail — optional / for replay & km recap)
```sql
CREATE TABLE location_history (
  id              BIGINT NOT NULL AUTO_INCREMENT PRIMARY KEY,
  company_id      CHAR(36) NOT NULL,
  courier_id      CHAR(36) NOT NULL,
  route_id        CHAR(36),
  latitude        DOUBLE NOT NULL,
  longitude       DOUBLE NOT NULL,
  heading         FLOAT,
  speed_kmh       FLOAT,
  recorded_at     DATETIME NOT NULL,
  KEY idx_loc_courier_time (courier_id, recorded_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
-- High volume: prune/aggregate older rows; only keep what analytics needs.
```

### messages (chat)
```sql

  id              CHAR(36) NOT NULL PRIMARY KEY,
  company_id      CHAR(36) NOT NULL,

  sender_id       CHAR(36) NOT NULL,

```

### notifications
```sql
CREATE TABLE notifications (
  id              CHAR(36) NOT NULL PRIMARY KEY,

  user_id         CHAR(36) NOT NULL,
  type            ENUM('new_task','route_update','delivered','chat','approval','system') NOT NULL,
```

## 3. Tenancy & Integrity Notes
- **Every** SELECT/UPDATE/DELETE on tenant tables filters `company_id` (Laravel global scope; Go middleware-injected context). Code review blocks any tenant query without it.
- UUIDs generated in app (Laravel `HasUuids`, Go `google/uuid`) — store as `CHAR(36)`. `catering_code` = random URL-safe token (not serial); rotated on demand.
- Spoilage values (`spoilage_risk`, `minutes_until_spoilage`) are **persisted from app.py output** — never recomputed in the UI; display only.
- `routes.depot_*` is a snapshot so editing company depot later doesn't rewrite history.
- Analytics: sales from `orders`/`subscriptions`; courier recap from `routes`/`route_assignments` (km, counts, on-time = `delivered_at <= deadline_at`). Lazy-load last 30 days by default (NFR-6).
- Cancel-mid-route edge: set `route_assignments.status='skipped'`, recompute `routes` totals, emit `route_update` notif to admin.
- Live map read = single-row lookup from `current_locations` per courier (upsert on each ping); `location_history` is optional append-only for replay/recap.

## 4. Migrations ownership
**Laravel owns the schema** (Eloquent migrations): `$table->uuid('id')->primary()`,
`$table->enum(...)`, `$table->json(...)`, `$table->foreignUuid(...)->constrained()`.
Laravel `.env`: `DB_CONNECTION=mysql`, point at Laragon's MySQL (`127.0.0.1:3306`).
**Go** reads/writes the realtime + routing tables on the **same MySQL DB** via
`database/sql` + `go-sql-driver/mysql` (or `sqlx`/`sqlc` MySQL). Single DDL source = Laravel migrations.

## 5. Local setup (Laragon)
1. Laragon → Start All (MySQL on `3306`). Open **HeidiSQL** (Menu → Database).
2. Create database `logieat` (utf8mb4). Set Laravel `.env` to it; run `php artisan migrate`.
3. Go service uses the same DSN: `user:pass@tcp(127.0.0.1:3306)/logieat?parseTime=true&loc=UTC`.
> Redis (realtime) is separate — add Redis in Laragon or run Memurai (Windows). Go + Python + OSRM run outside Laragon (see ARCHITECTURE §8).
